Follows From

Follows From is built around a single motion: questions move backward toward what would ground a trace, and consequences move forward from whatever model survives that question.

Trace → Question → Model → Answer → Reason → Check
Read the manifesto

The three doors

The Library

The side of questions: traces, comparison, legibility, partial knowledge, and reading from within.

The Path

The side of unfolding: origin, evolution, dynamics, inheritance, consequence, and becoming.

The Method

The craft of turning questions into trustworthy artifacts built as Answer · Reason · Check.

Why this name

Follows From names the standard the project cares about most: every claim should show what it follows from, and every artifact should make that line readable.

A trace invites a backward question — what would explain this? A model then moves forward — what follows from it? The name holds both directions in ordinary language without sounding ornamental.
